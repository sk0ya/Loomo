# 01. 概要とアーキテクチャ

> 章番号（§N）は設計書全体で通し。他章の §N がどのファイルかは [README](README.md) の「§→ファイル対応表」を参照。

## 1. 概要

ローカルの開発ワークスペースを対象に動作する **AIエージェント搭載のデスクトップアプリ**（C# / WPF）。
ユーザーは自然言語で指示を出し、AIエージェントが **Terminal（コマンド実行）** と **Editor（ファイル表示・編集）** を操作しながらタスクを遂行する。

> **本アプリはローカルLLM専用**（2026-06-04 決定）。推論は **in-process のローカル推論エンジン**で行い、
> クラウドAI（Claude / OpenAI / GitHub Copilot 等）にも外部サーバ（旧 Ollama）にも接続しない。設計の狙いは
> プライバシー・オフライン動作・構成の単純さ。**現行の正本は §25**（ONNX Runtime GenAI ＋ llama.cpp の
> in-process 推論／4 ツール構成）。クラウドプロバイダ前提の旧記述、および外部 Ollama サーバ前提の §22 は
> §25 に読み替えること。

### コアコンセプト
- AIエージェントは UI を「人間と同じ道具」として使う。すなわち FolderTree で対象を把握し、Terminal で実行し、Editor で表示・編集する。
- UI の各ペインは **疎結合なサービス + メッセージング** で連携し、AIエージェントは「ツール（Tool）」を通じてそれらを駆動する。
- モダンな見た目（ダーク基調・フラット・VS Code 系の操作感）と、機能を後から足しやすい拡張構造を両立する。

## 2. 技術スタック

| 領域 | 採用 | 理由 |
|------|------|------|
| ランタイム | .NET 9（`net9.0` / `net9.0-windows`） | 現行実装のターゲット |
| UIフレームワーク | WPF | 要件指定。リッチなデスクトップUIに最適 |
| アーキテクチャ | MVVM | テスト容易・View/ロジック分離 |
| MVVM基盤 | CommunityToolkit.Mvvm | `ObservableObject` / `RelayCommand` / `WeakReferenceMessenger` |
| DI / 起動 | Microsoft.Extensions.Hosting + DependencyInjection | サービス登録・ライフサイクル管理・設定・ロギングを統一 |
| ロギング | Microsoft.Extensions.Logging.Abstractions | Core 側は抽象に留める |
| Terminal | `sk0ya.Terminal.Controls` | `TerminalTabView.RunCommandAsync` で実行、`SetColorTheme`/`SetFont` で配色・フォントを反映。既定シェルは `pwsh.exe`（バージョンは `src/Loomo.Services/Loomo.Services.csproj` の 1 箇所ピン） |
| Editor | `sk0ya.Editor.Controls` / `.Defaults` | `VimEditorControl`・仮想ドキュメント・LSP・CLI フォーマッタを利用（バージョンは同 csproj ピン） |
| Browser | `Microsoft.Web.WebView2` | ブラウザペイン（Web タブ）。お気に入りアイコン取得・ナビゲーション |
| AIクライアント | ローカルLLM（in-process／HTTP なし） | `IAiClient`＝`OnnxGenAiClient` 単一実装。推論は `ILocalInferenceEngine`（ONNX Runtime GenAI ／ llama.cpp を `modelPath` で振り分ける `LocalInferenceRouter`）。詳細は §6.2 / §25 |
| 設定保存 | `%APPDATA%/Loomo/settings.json` + DPAPI | モデルパス等を保存。APIキー（ローカル推論では不要）を CurrentUser スコープで暗号化。旧 `baseUrl`/`SystemPrompt` は無視し、旧 `thinkingEffort` は現行 `thinking` へ移行する |
| モデル保存 | `%APPDATA%/Loomo/models/` | `ModelDownloadService` がダウンロードした ONNX（CPU int4）モデルを格納 |
| ワークスペース状態 | `%APPDATA%/Loomo/workspaces.json` | 複数ワークスペース・ペインレイアウト・各ペインのタブを保存 |

> AIプロバイダは `IAiClient` 抽象の背後に隠す。**実装は in-process のローカル推論（`OnnxGenAiClient`）のみ**。
> 推論エンジンは `ILocalInferenceEngine` の背後で ONNX Runtime GenAI と llama.cpp（GGUF）を切替（§25）。
> クラウド接続も外部サーバ（Ollama）も対象外。

## 3. 全体アーキテクチャ

レイヤー構成（依存は上→下の一方向）:

```
┌─────────────────────────────────────────────────────────┐
│  Presentation (WPF)                                       │
│   Views (XAML) ── ViewModels ── Behaviors / Converters   │
└───────────────┬─────────────────────────────────────────┘
                │  Messenger（イベント） / DI
┌───────────────▼─────────────────────────────────────────┐
│  Application / Agent Core                                 │
│   AgentOrchestrator（エージェントループ）                 │
│   ToolRegistry / IAgentTool 群                            │
│   ApprovalService（実行承認）/ ConversationStore          │
└───────────────┬─────────────────────────────────────────┘
                │  サービス抽象（インターフェース）
┌───────────────▼─────────────────────────────────────────┐
│  Services / Adapters                                      │
│   ITerminalService → sk0ya.Terminal                      │
│   IEditorService   → sk0ya.Editor                        │
│   IWorkspaceService→ ファイルシステム / FolderTree状態     │
│   IAiClient        → OnnxGenAiClient（in-process ローカル推論）│
└──────────────────────────────────────────────────────────┘
```

UI型に依存しない再利用可能な純粋ロジックは App に置かない。ファイル判定・リンク解決は
`Loomo.Core/Files`、Markdown テーブル変換は `Loomo.Core/Markdown`、Git 出力解析は
`Loomo.Core/Git` に置く。端末向けスクリプト生成のようにOS／I/Oへ接続する処理は
`Loomo.Services` に置き、App は表示・入力・画面遷移へ集中する。

### 連携の要点
- **View 同士は直接参照しない。** すべて `WeakReferenceMessenger` のメッセージ、または共有サービス経由でやり取りする。
- AIエージェントは UI を直接触らず、**ツール → サービス → （メッセージ）→ View** という経路で操作する。これにより「AIが操作する」「人間が操作する」が同じ経路に集約され、テスト・記録・再現が容易になる。

```

│  │   ├─ Views/
│  │   │   ├─ ShellWindow.xaml(.cs)← ルートレイアウト＋ペインツリー構築（code-behind が中核）
│  │   │   ├─ FolderTreeView
│  │   │   ├─ TabsView             ← Terminal/Editor/Browser タブの横断一覧
│  │   │   ├─ SessionsView
│  │   │   ├─ SettingsView         ← AIプロバイダ・安全設計
│  │   │   ├─ AppearanceView       ← カラーテーマ・アクセントカラー
│  │   │   └─ AiBarView            ← AI ペインの中身（会話履歴・ツールカード・入力）
│  │   ├─ ViewModels/             ← Shell / FolderTree / WorkspaceList / Tabs / Sessions / Settings / Appearance / AiBar
│  │   ├─ Services/                ← UiApprovalService / WorkspaceStateStore / TabIconService / ThemeManager
│  │   ├─ Themes/                  ← Palette.<Theme>.xaml（Dark/Light/SolarizedDark/Nord/HighContrast）/ Controls.xaml
│  │   └─ Converters/
│  │
│  ├─ Loomo.Core/           ← ドメイン（UI非依存・テスト対象の中核）
│  │   ├─ Agent/                  ← AgentOrchestrator・会話モデル
│  │   ├─ Tools/                  ← IAgentTool・ToolRegistry・各ツール
│  │   └─ Abstractions/           ← ITerminalService 等のインターフェース
│  │
│  ├─ Loomo.Ai/             ← IAiClient 実装（OnnxGenAiClient）＋推論エンジン（ONNX/llama.cpp）と設定保存

│  └─ Loomo.Services/       ← sk0ya.* アダプタ / FileSystem 実装
│
└─ tests/
   └─ Loomo.Tests/         ← Core / Tools のユニットテスト
```

> ポイント: **Loomo.Core は WPF を参照しない。** エージェントの頭脳とツールはUIから独立させ、単体テスト可能にする。

## 8. 拡張性の設計

| 拡張軸 | 仕組み |
|--------|--------|
| サイドバーパネル追加 | 現状は `ShellViewModel.ActivePanel`（`SidebarPanel` enum）による固定6パネル（Explorer/Search/Tabs/Git/Sessions/Pegboard）。Settings/Appearance は中央オーバーレイへ分離（§4/§26）。`IPanelContribution` 方式は将来の拡張案 |
| ペイン配置 | レイアウトツリー（`PaneNode`）でドラッグ並べ替え・分割・リサイズ・表示切替に対応（§21）。新ペイン種別は `PaneKind` 追加＋ホスト要素追加で拡張 |
| AIツール追加 | `IAgentTool` 実装 + DI登録（ToolRegistryが自動収集） |
| AI推論バックエンド | `IAiClient` 実装は `OnnxGenAiClient` のみ。推論エンジンは `ILocalInferenceEngine` 越しに ONNX/llama.cpp を `modelPath` で切替（**ローカル推論のみ対象**・§25） |
| テーマ | パレット（`Palette.*.xaml`）を分離し `ThemeManager` で実行時切替。Dark/Light/SolarizedDark/Nord/HighContrast の5種＋アクセントカラー上書き。新テーマは `AppTheme` 追加＋同名パレット追加＋外観パネルに1行追加で拡張 |
| UI文字サイズ | アプリ全体の文字はトークン方式（`Typography.xaml` の `Fs*` シード値、本文 `Fs13` を基準に比率化）。`UiFontManager` が外観パネルの設定（既定16px基準）から `Fs*` を再計算し `DynamicResource` 経由で全UIへ反映。生 `FontSize` 直書きは禁止。エディタ／ターミナルのフォントサイズはこれと非連動（各コントロール側の設定） |
| コマンド/ショートカット | 将来 `ICommandContribution` でコマンドパレット・キーバインド登録 |
| 設定 | `%APPDATA%/Loomo/settings.json`。APIキーは DPAPI で暗号化 |

> 「ドッキング自由化」は AvalonDock を導入せず、独自のレイアウトツリー＋ `PaneHost` 動的構築で実装した（§21）。
> WebView2 を含むペインを再ペアレントしても生存させられる軽量な方式を選んだ。

## 9. 状態管理・DI・ライフサイクル

- `App.xaml.cs` で `Host` を構築し、サービス／ViewModel／ツールをすべてDI登録。
- スコープ：サービスは概ね Singleton（Terminal/Editor/Workspace/AiClient/ToolRegistry）。ViewModelは用途に応じ Singleton/Transient。
- 会話状態は `ConversationStore`（Singleton）に集約。AIセッションの保存/復元に対応（ActivityBarの「AIセッション」から履歴閲覧）。
