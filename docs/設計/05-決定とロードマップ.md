## 12. 決定事項 / 未決事項

### 決定済み（2026-05-31）
1. **.NET バージョン**：**.NET 9**（`net9.0` / WPFは `net9.0-windows`）。
2. **sk0ya.Terminal / sk0ya.Editor**：**WPFコントロールとして直接埋め込む**。両パッケージはユーザー本人が開発しており、必要なら **API側を変更可能**。→ アダプタ層は「理想的なサービスIF」を先に定義し、コントロール側を歩み寄らせる方針。
3. **AIプロバイダ**：**ローカルLLM専用**（2026-06-04 決定）。`IAiClient` 抽象 + `IAiClientFactory` は残すが
   実装は `OnnxGenAiClient` のみ。推論は in-process（ONNX Runtime GenAI ／ llama.cpp、§25）で外部サーバ無し。
   クラウドプロバイダ（Claude / OpenAI / Copilot）も外部 Ollama も対象外。
4. **ペイン配置**：当初の固定配置（中央=Terminal/右=Editor）を廃し、ユーザーが自由に並べ替え・分割できる
   ツリー構造レイアウトへ刷新（§21）。既定は `[Editor | Browser] / [Terminal] / [AI]`。
5. **承認の既定**：初期は承認制。自動承認は設定でオプトイン。

### AIプロバイダ設計（追補・2026-06-29 更新）
```
IAiClient（抽象）
 └─ OnnxGenAiClient（プロバイダ Local・in-process）
      └─ ILocalInferenceEngine ── LocalInferenceRouter（modelPath で振り分け）
            ├─ OnnxGenAiEngine （genai_config.json を持つフォルダ → ONNX Runtime GenAI / CPU int4）
            └─ LlamaCppEngine  （.gguf → llama.cpp / LLamaSharp）
```
- 推論は in-process のローカル推論のみ。`IAiClientFactory` はこの単一実装を返す（HTTP も外部サーバも無い）。
- 旧 Ollama（`OllamaProtocol`/`OllamaClient`/`OllamaLauncher`）、および Stub / Claude / OpenAI互換 / Copilot
  クライアントと共用の `OpenAiProtocol` はいずれも廃止（§22→§25）。詳細は §6.2 / §25。

## 19. 今後の実装構想（2026-06-01 ブレインストーム）

> ここから先は **未確定の構想** メモ。実装済みと明記した項目を除き、優先度・実現性は今後精査。
> 大きな方向性は「**AIエージェントの観測・制御**」「**マルチエージェント**」「**ワークスペース／レイアウトの自由化**」
> 「**集合知（ナレッジ）化**」の4本柱。

### 19.1 エージェントの観測性（Observability）
- **全ログの永続保存**: AIエージェントのやり取り（プロンプト・ツール呼び出し・結果・思考）をすべて構造化して保存する。
  現状のセッション保存（§16）は会話の往復のみ。これを拡張し、ツール実行の引数・出力・所要時間・トークン等まで記録する
  詳細ログ層を別途設ける（セッションJSONとは分離し、解析しやすい形式に）。
- **ログ解析**: 保存したログを後から解析できるようにする（どのツールがどれだけ呼ばれたか、失敗パターン、所要時間など）。
- **動作中エージェントの可視化**: 実行中に「いまどのエージェントが動いているか」を UI で分かるようにする
  （後述のマルチエージェントと連動）。
- **プレイバック**: 保存したログ／セッションを再生（タイムライン的に追体験）できるようにする。
  ※ §3 の「AIも人間も同じ経路で操作する」設計が、記録・再現を容易にする前提として効いてくる。

### 19.2 マルチエージェント
- **子エージェントの設定**: 親エージェントが子（サブ）エージェントを構成・起動できるようにする。
- **稼働状況の可視化**: 複数エージェントが動くとき、どれがアクティブかを UI で識別できるようにする（19.1 と共通）。

### 19.3 ワークスペース／プロジェクト管理
- **ワークスペースの切替**: 複数ワークスペースをタイトルバーの ComboBox で切り替えられる。→ **完了（§21）**
  各ワークスペースごとに Terminal/Editor/Browser のタブ群とペインレイアウトを `workspaces.json` に保存・復元する。
- **サイドバーでのプロジェクト管理**: 個別ワークスペースだけでなく、**全体のプロジェクト群を横断管理**する
  ビューをサイドバーに持たせる構想（一覧 UI は未着手。現状は ComboBox 切替）。

### 19.4 FolderTree 強化
- **ignore の非表示**: `.gitignore` 等で無視されるファイルを隠すオプション。→ **完了（2026-06-01）**
  FolderTree 上部のコンパクトなツールバーにトグルとして配置し、`git check-ignore --stdin` によるバッチ判定で表示対象を絞り込む。
- **差分のみ表示フィルター**: 変更があったファイルだけを表示するフィルター。→ **完了（2026-06-01）**
  `git status --porcelain -z --untracked-files=all` を用いて変更ファイルとその親ディレクトリだけを表示する。
- **自動更新**: ワークスペース配下を `FileSystemWatcher` で監視し、作成・削除・変更・リネーム時に debounce 付きで Git 状態とツリーを再読込する。→ **完了（2026-06-01）**
- **Vim 風キーボード操作＋インクリメンタル検索**: → **完了（2026-06-01）**
  j/k/h/l/gg/G でのキーボード移動と `/` 検索（逐次ヒット・n/N 巡回）を `FolderTreeView` の code-behind に実装（§4）。
- **汎用フィルター機能**: 名前・種別などでツリーを絞り込む（`/` 検索は移動用で、表示の絞り込みは別途。未着手）。

### 19.5 レイアウト／タブの自由化（ツールとしての強み）
- **レイアウトをユーザーが決める**: 固定 Grid から、ユーザーがペイン配置を自由に決められる方式へ。→ **完了（§21）**
  レイアウトツリー＋ドラッグ並べ替え・分割・リサイズ・表示切替で実現（AvalonDock は不採用）。
- **Editor・ターミナル・ブラウザを自由配置**: これらを自由に開閉・配置できるのがこのツールの強み。→ **完了（§21）**
  - **Web タブの追加**: ブラウザペイン（Web タブ）を新設し、Editor／Terminal と並べる。→ **完了（WebView2・§21）**
  - **タブの AI 管理**: 各ペインがタブを持つが、その開閉・整理を **AI が管理**できると良い（タブ基盤は完成、AI ツール化は未着手）。
  - **AI への「開く」指示**: 「これを開いて」と AI に指示して、対象（ファイル/URL/マップ等）を適切なペインで開かせる（未着手）。
- **全タブのサイドバー集約**: ターミナル・ブラウザ・Editor のタブをすべてサイドバーに集める。→ **完了（Tabs パネル・§21）**
  - **アプリのカスタム埋め込み**: 任意アプリを埋め込めるようにする。Windows 側で DLL を作れば実現できそう（未着手）。
- **画面分割**: ターミナル／Editor／ブラウザはそれぞれ分割表示可能に。→ **完了（ツリーの行/列分割で任意ペインを並置・§21）**
- **ペイン全画面表示**: **F11** で現在のペインだけをモニター全体（タスクバー領域を含む）に表示し、もう一度押すとウィンドウ位置・サイズ・ペイン配置へ戻る。

### 19.6 集合知（ナレッジ）化
- **AI へのインプットがそのまま集合知に**: ユーザーが AI に与えた情報を蓄積・再利用できる知識ベースにする。
  - Web のリンク収集、**ブックマークの更新**。
  - **マインドマップの特定マップを開く**といった連携も視野。

### 19.7 ランチャー
- **ランチャーからの指示**: アプリ外（ランチャー）から AI へ指示を投げられるようにする。
