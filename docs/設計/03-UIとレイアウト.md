## 4. UIレイアウト（現行：ツリー構造ペイン）

採用: **カスタムタイトルバー + ActivityBar/Sidebar + 自由配置のペイン群（AI もペインの1つ）**

> 旧案（全幅下部AIバー）から刷新。メイン領域は固定 Grid ではなく **レイアウトツリー**で表現し、
> ユーザーがペインをドラッグで 2D 並べ替え・分割・リサイズ・表示切替できる（§21）。AI は専用バーではなく
> 1つのペインになった。

### レイアウト実装方針
- メイン領域は **レイアウトツリー**（`PaneNode` = リーフ `PaneLeaf`／スプリット `PaneSplit`）で保持し、`PaneHost`（Grid）を
  `RebuildPaneLayout` がツリーから動的に組み立てる。スプリットごとに行/列 Grid を生成し、ペイン本体を再ペアレントする
  （同一ウィンドウ内なので WebView2 等は生存）。詳細は §21。
- 各ペインは **ドラッグで 2D 並べ替え**（ドロップ先の辺を半透明プレビューで表示。WebView2 の airspace を越えるため
  最前面 `Popup` で描画）、**GridSplitter でリサイズ**、**タイトルの「—」で非表示**（タイトルバーのトグルで再表示）。
- 既定レイアウトは `[Editor | (EditorSupport:隠) | Browser]`（上段）/ `[Terminal]` / `[AI]`（縦積み）。
- **2つの表示モード**：通常の **タイル（レイアウトツリー）** と、1ペインを全面に立てる **ソロ（舞台＋袖＋俯瞰）**
  をタイトルバーで切替（§26）。ソロはツリー（`_root`）に触れず表示だけ差し替えるので、レイアウトへ戻すと元配置に復帰。
- 名前付きレイアウトの**保存・巡回**（Ctrl+T）に対応（`ShellWindow.Layouts.cs`）。
- サイドバー切替は `ShellViewModel.ActivePanel`、開閉は `IsSidebarVisible`（列幅と GridSplitter を code-behind で切替）。
- 配色は `Themes/Palette.<Theme>.xaml`（ブラシのみ・Dark / Light / SolarizedDark / Nord / HighContrast）に分離し、
  角丸・余白・スタイル類は `Themes/Controls.xaml` に集約する。スタイルは色を `DynamicResource` で参照するため、
  `ThemeManager` がマージ辞書のパレットを差し替えるだけで全 UI に即時反映される。
- アクセントカラーは `ThemeManager` が `Application.Resources` 直下に `Accent` / `AccentHover` / `AccentFg` を置いて
  パレット定義を上書きする（マージ辞書より直下キーが優先。`AccentHover` は自動で明色化、`AccentFg` は輝度から白/黒を選択）。
  空ならテーマ既定へ戻る。
- テーマ／アクセントの選択は **外観（Appearance）パネル**（ActivityBar の 🎨）で行い、AIプロバイダ設定（⚙）とは分離。
  どちらも `settings.json` に永続化される。

## 21. ペインレイアウト刷新・ブラウザ・複数ワークスペース（2026-06-01 実装完了）

> §4 を全面刷新した実装の記録。「全幅下部 AIバー＋固定 Grid」を廃し、**メイン領域をレイアウトツリーで保持して
> 動的に組み立てる**方式へ移行。あわせて Browser ペイン・各ペインのタブ・複数ワークスペースを導入した。
> 中核ロジックは `ShellWindow.xaml.cs`（code-behind）に集約している。

### 21.1 レイアウトツリーと動的構築
- ツリーは `PaneNode`（基底）＝ `PaneLeaf`（1ペイン）／ `PaneSplit`（行=上下 or 列=左右の入れ子）で表す。
  ルートを `_root` に保持し、`RebuildPaneLayout` が **スプリットごとに Grid を生成**して `PaneHost` を組み直す。
  ペイン本体（`TerminalPane`/`EditorPane`/`BrowserPane`/`AiPane`）は XAML に1つだけ定義し、再構築のたびに生成した
  Grid へ **再ペアレント**する（同一ウィンドウ内なので WebView2/Terminal の HWND は生存）。
- **正規化** `Normalize`：空スプリット除去・子1個のスプリット畳み込み・同方向スプリットのフラット化。
- **既定レイアウト** `ApplyDefaultLayout`：`[Editor | Browser]`（列）/ `[Terminal]` / `[AI]`（行）。
- 列/行トラックには可視な子だけを割り当て、間に `GridSplitter` を差す。`GridSplitter` 操作後の star 比率は
  `CaptureLayoutSizes`→`CaptureNode` で各ノードの `Weight` に取り込み、保存・再構築で維持する。

### 21.2 ドラッグ並べ替え（スナップ風プレビュー）
- ペインタイトルを掴んでしきい値以上ドラッグすると `BeginPaneDrag`。**WebView2 の airspace を越えて最前面に描く
  ため、透明 `Popup`＋`Canvas`** を被せ、その上でマウス追跡＆プレビュー（ドロップ先ペインの枠＋落とす辺の半分を
  半透明で塗る）を行う。
- `HitTestCell` でカーソル下のペイン矩形を求め、`NearestZone` で最も近い辺（Left/Right/Above/Below）を判定。
  ドロップで `MovePane`→`RemoveNode`＋`InsertRelative`：望む方向が対象の親スプリットと一致すれば兄弟挿入、
  異なれば対象を新スプリットで包む（＝列の片方だけを上下分割できる）。

### 21.3 ペインの表示切替
- タイトルの「—」(`OnHidePane`) でそのペインを隠す。ActivityBar のトグル（▦/✎/🌐/🤖, `OnTogglePaneVisibility`）で
  再表示。非表示はツリーから消さず `PaneLeaf.Hidden` を立てるだけなので、**再表示で元の位置・比率に戻る**。
  可視ペインが1枚のときは最後の1枚を隠さない。
- **ペイン間フォーカス移動（Ctrl+W → h/j/k/l, vim 風）**: `Ctrl+W` でプレフィックス待ち状態に入り、続く
  `h`/`j`/`k`/`l` でフォーカス中ペインの左/下/上/右の隣接ペインへフォーカスを移す（`FocusPaneInDirection`）。
  方向は `HitTestCell`/`NearestZone` と同じ矩形判定でフォーカス中ペインに最も近い隣接ペインを選ぶ。
- **前面表示の配置（`PlacePaneByBehavior`、2026-07-06）**: 「AI に聞く」「ブラウザで調べる」「差分を開く」等の
  結果表示・ファイルを開く・袖ミニチュアのクリックは、対象ペインを前面へ出す操作を **共通関数
  `PlacePaneByBehavior` に集約**し、設定 `AiSettings.PaneOpenBehavior`（外観カテゴリのコンボボックス、
  既定 `Main`）で配置を切替えられる：
  - `Main`（従来挙動）：左上のペインと入れ替える。
  - `Sub`：右上のペインと入れ替える（上段が横1枚しか無ければ右へ追加）。右上（サブ）の判定は矩形の実測値
    ではなく**構造ベース**（`PaneLayoutTree.TopRow`/`RightmostVisibleLeaf`/`LeftmostVisibleLeaf`）——矩形が
    未確定なタイミングで下段を誤って選んでしまう不具合を避けるため。
  - `Loop`：常にサブ側に表示する。操作の起点がサブペイン自身なら、現在のサブをメインへ繰り上げてから表示する。
  - **対象ペインが既に画面に可視なら、いずれのモードでもレイアウトの入れ替え自体は行わない**
    （フォーカス移動・内容表示＝AI結果/差分/ファイル内容は従来どおり行う）。非表示のときだけ設定どおりに配置する。
  - `settings.json` は `null` を `Main` として後方互換で読む。

### 21.4 タブ（Terminal / Editor / Browser）
- 各ペインは複数タブを持つ。本体（`TerminalTabView`/`VimEditorControl`/`WebView2`）は専用の ContentHost に重ね、
  アクティブなものだけ `Visible` にする。タブ見出しは各ペイン上部のタブストリップ（横スクロール対応）に表示。
- サイドバーの **Tabs パネル**（`TabsViewModel`）が3種のタブを横断一覧化。クリックで該当タブをアクティブ化、×で閉じる
  （`TabActivated`/`TabCloseRequested` を code-behind が購読）。タブのアイコンは `TabIconService` が供給：
  ファイルは Shell アイコン（`SHGetFileInfo`）、ターミナルは pwsh/powershell の実行ファイルアイコン、
  ブラウザは WebView2 の favicon（`GetFaviconAsync`、取得失敗時は Edge アイコン→ベクターのフォールバック）。

### 21.5 Browser ペイン（WebView2）
- `Microsoft.Web.WebView2` を新規ペインとして追加。アドレスバー（URL/検索クエリを `NormalizeBrowserAddress` で解釈、
  localhost は http、空白を含めば Google 検索）・戻る/進む/更新・favicon・複数タブに対応。既定 URL は Google。
- `NavigationCompleted`/`FaviconChanged` でタブのタイトル・アイコン・アドレス表示を更新。

### 21.6 複数ワークスペースと永続化
- タイトルバーの ComboBox（`WorkspaceListViewModel`）でワークスペースを切替、「＋」で `OpenFolderDialog` から追加。
- 状態は `WorkspaceStateStore`（`%APPDATA%/Loomo/workspaces.json`）に保存。`WorkspaceSnapshot` が
  RootPath・各ペインのタブ（`TerminalTabSnapshot`/`EditorTabSnapshot`/`BrowserTabSnapshot`）・アクティブタブ・
  ペインレイアウト（`PaneNodeSnapshot`）を保持。`SaveActiveWorkspaceSnapshot` がタブ操作・レイアウト変更・終了時に
  `CaptureInto` でスナップショットを更新して書き出す（即時永続化）。
- ワークスペース切替 `SwitchWorkspaceAsync` は現ペインのタブを Detach→対象の保存済みタブ/レイアウトを復元。
  各ワークスペースのライブなタブ集合はメモリ上（`_terminalWorkspaces` 等）にもキャッシュし、再切替で素早く Attach する。
- フォルダを開くと FolderTree のルート・エージェントの作業ディレクトリ（`TerminalService.SetWorkingDirectory`）も同期。

### 21.7 カスタムタイトルバー
- `WindowStyle=None` + `WindowChrome`。最小化/最大化(復元)/閉じるを自前ボタンで実装。`WindowStyle=None` のままだと
  最大化でタスクバーを覆うため、`WM_GETMINMAXINFO` を処理して最大化サイズを**モニタのワーク領域**に収める。

### 21.8 主な関連ファイル
- `src/Loomo.App/Views/ShellWindow.xaml(.cs)`：レイアウトツリー・ドラッグ・タブ・ブラウザ・ワークスペース切替・タイトルバー。
- `src/Loomo.App/Services/WorkspaceStateStore.cs`：`WorkspaceState`/`WorkspaceSnapshot`/`PaneNodeSnapshot` 等の DTO と保存。
- `src/Loomo.App/Services/TabIconService.cs`：Shell/favicon アイコン供給。
- `src/Loomo.App/ViewModels/{TabsViewModel, WorkspaceListViewModel}.cs`：タブ横断一覧・ワークスペース一覧。

## 24. 部屋の操作統一・袖の周辺視野化・素材の流れ・復元の完全性（2026-06-13 実装完了）

> 方向性「部屋（Room）としての完成度」の4本を同時に実装。AIに依存しない、道具としての進化。

### 24.1 袖＝周辺視野（ターミナル活動バッジ・OSC133）

**目的**: 長いビルド/テストを袖のターミナルに置いたまま、終わったこと（と成否）を目の端で気づける。

- **ライブラリ**: `sk0ya.Terminal.Controls` **1.0.8** が `TerminalTabView.ShellCommandActivity`
  公開イベントを追加（OSC133 A/B/C/D を `ShellCommandPhase` ＋ `ExitCode` で通知。1.0.7 の
  pwsh 自動注入により設定不要で発火する）。Loomo 側は `ShellWindow.PaneActivity.cs`。
- **表示**: 袖/俯瞰のターミナルカード右上にバッジ。実行中（C 受信）＝「● 実行中」（Accent色）、
  未確認の完了＝「✓ 完了」（緑）/「✗ 失敗 {exitCode}」（赤）。
- **既読の流れ**: ターミナルが目に入っている間（舞台に立っている・タイルで可視）の完了は記録しない。
  未確認結果はターミナルが舞台に立った時点（`SetStagePane`）/タイル復帰時に消える。
- **偽バッジ対策**: C（実行開始）を見ていない D は無視する（シェル統合スクリプトの初期化や
  プロンプト再描画が D を発し、起動直後に偽の「✓ 完了」が出るのを実機で確認・修正済み）。
- 複数タブは集約（実行中 ＞ 失敗 ＞ 成功）。タブクローズで状態破棄。

### 24.2 コマンドパレット（部屋全体の操作統一）

**目的**: 部屋の全操作に名前を付けて1つの口から呼べるようにする（§19.5「コマンド/ショートカット」の実装）。

- **Ctrl+Shift+P** または **Ctrl+W p** で開くオーバーレイ（`ShellWindow.CommandPalette.cs`＋XAML、
  絞り込みは `Services/CommandPalette.cs` の `PaletteFilter`＝純ロジック・テスト済み）。
- コマンド: ステージ切替/俯瞰、移動（各ペイン・ステージ中は舞台転換）、ペイン表示切替、
  新規タブ（Terminal/Editor/Browser）、コンポーザ（開閉/実行/ピン）、ペグボード各操作、
  サイドバーパネル切替、ワークスペース切替（動的）。開くたびに現在状態から再構築。
- 絞り込み: 空白区切り AND・前方一致 ＞ 部分一致 ＞ 飛び石一致のランキング。↑↓/Enter/Esc。

### 24.3 素材の流れ（ペグボード⇄コンポーザ⇄ターミナル⇄エディタ）

**目的**: §23 の作業台（コンポーザ）と道具掛け（ペグボード）を孤立パネルから「ペイン間を流れる素材」へ。

- **ペグボード→ターミナル**: カード右クリック「ターミナルへ送る」＝可視ターミナルのプロンプトへ入力のみ
  （FolderTree「ターミナルにセット」と同じ流儀）。**複数行はコンポーザへ流す**（プロンプト暴発防止）。
- **ペグボード→コンポーザ**: 「コンポーザへ挿入」＝本文末尾へ追記して表示。
- **コンポーザ→ペグボード**: ヘッダー「📌 ペグボードへ」で本文をピン（組んだコマンドの再利用）。
- **エディタ→ペグボード**: パネルの「📌 エディタの選択をピン」（`SelectedText`）。
- **ドラッグアウト**: カードはテキスト（file はファイル参照も）としてドラッグ可能（外部アプリへも落とせる）。

### 24.4 復元の完全性（「場所」としての信頼）

- `EditorTabSnapshot` に **CaretLine/CaretColumn/ScrollRatio** を追加。復元は `NavigateTo`（即時）＋
  `ScrollToVerticalRatio`（Loaded 優先度へ遅延・ベストエフォート。取れなくてもカーソル可視化が補う）。
- `WorkspaceSnapshot` に **ComposerVisible/ComposerHeight** を追加（開いたまま離れたら開いたまま戻る。
  復元順序に注意: `SetComposerVisible(false)` が高さを退避するため、高さ反映は表示切替の後）。
- `StageSnapshot` に **Overview** を追加（俯瞰を開いたまま離れたら俯瞰のまま戻る）。
- テスト: `ComposerPegboardTests`（スナップショット往復・素材の流れイベント）、`PaletteFilterTests`。

## 26. ペイン構成・サイドバー・設定オーバーレイ・ステージモードの現況（2026-06 整理）

> §4 の「ツリー構造ペイン」を土台に、ペイン種別とサイドバーが大きく増えた。**ペイン/サイドバーの現行正本は
> §4 ＋本節**。実装は `ShellWindow.*.cs`（partial 多数）と `ShellViewModel`／各ペイン VM に分散している。
> 列挙の真実は `WorkspaceStateStore.PaneKind`（ペイン）と `ShellViewModel.SidebarPanel`（サイドバー）。

### 26.1 ペイン種別（`PaneKind`・9種）
レイアウトツリーのリーフに乗るペイン。本体は XAML に1つずつ定義し、再構築のたびに生成 Grid へ再ペアレントする（§21.1）。
`InitializePanes` が `_paneElements[PaneKind.*]` を登録し、`ShellWindow.PaneLayout.cs` が組み立てる。

| `PaneKind` | 中身 / ホスト | 備考 |
|------------|---------------|------|
| `Terminal` | `TerminalTabView`（既定 pwsh）・複数タブ | 下部に**コンポーザ**をはめ込み可（§23.2）。AI 実行は独立プロセス（§13） |
| `Editor` | `VimEditorControl`・複数タブ | LSP・CLI フォーマッタ・Git 統合あり（CLAUDE.md 参照） |
| `EditorSupport` | Markdown 等のプレビュー（WebView2） | **既定 Hidden**。md 等を開くと Editor の右隣に自動出現／ソロ時は舞台へ。発表（marp）モード切替あり。ヘッダーから **HTML/PDF エクスポート**（HTML は仮想ホスト依存を外して単体化＝`PortableHtml`、PDF は表示のまま `PrintToPdfAsync`）。JSON プレビューは **スキーマ検証**（`$schema`／兄弟 `*.schema.json` を解決、http(s) はキャッシュ取得）で違反を波線＋先頭バナー表示 |
| `Browser` | `WebView2`・複数タブ・アドレスバー | `web_search` ツールもこの可視タブを使う（§25.5） |
| `Ai` | `AiBarView`（会話履歴・ツールカード・入力欄・📊AI内訳） | エージェントループの表示面 |
| `Git` | `GitSessionViewModel`：コミットグラフ（`git log --graph`）・ブランチ一覧・rebase/merge/cherry-pick/reset 等。ヘッダーはブランチ切替（下記の共通コントロール）＋コミット絞り込みのみ。コミット詳細の変更ファイル一覧は `git show --stat` の統計行をリンク化（`CommitStatLinks`）し、クリックでリポジトリルート基準に解決して Editor で開く（リネームは新パスへ解決） | **既定 Hidden**。サイドバー Git パネルに収まらない込み入った操作を担う |
| `Diff` | `DiffSessionViewModel`：git 差分の色分け表示 | **既定 Hidden**。Git ペイン/パネルからの差分表示先 |
| `Trace` | `TraceSessionViewModel`：`traces/*.jsonl` を `TraceReader` で読み、ターン区切りタイムライン（ツール呼び出し・承認・`ai.usage` の load/prefill/decode）で表示・読み取り専用 | **既定 Hidden**。§20 観測性 Phase B/C 相当の実装面 |
| `Debug` | `DebugViewModel`（IDE：ビルド・テスト・デバッグ）。DAP（netcoredbg）連携、サブ VM＝Launch/Profiles/Tests/Inspection/Breakpoints/Attach | **既定 Hidden**。C#プロジェクトの無いワークスペースでは全経路（トグル/有効セッション/保存レイアウト/ソロ舞台）から除外。詳細は正本 §28 |

- 表示トグルは**タイトルバー**のトグル群（`OnTogglePaneVisibility`・Tag=PaneKind 名）。非表示はツリーから消さず
  `PaneLeaf.Hidden` を立てるだけ（§21.3）。新ペイン追加は `PaneKind` 追加＋`_paneElements` 登録＋XAML 本体＋
  `workspaces.json` 後方互換（旧ファイルにはリーフが無いだけ＝Hidden 後挿入 `InsertRelative`）。

### 26.2 サイドバー（`SidebarPanel`・7種）＋設定オーバーレイ
- **ActivityBar はサイドバー切替専用**になった（旧 §4 のペイン表示トグルはタイトルバーへ移動）。アイコン：
  上＝🗂Explorer/🔍Search/⚠Problems/📑Tabs/⎇Git/💬Sessions/📌Pegboard、下寄せ＝⌨キーボード設定・⚙設定。
- サイドバーは7パネルを排他表示（`Activate`＝同一再クリックで閉じる VS Code 風）。Git パネルは見えている間だけ
  ライブ監視（`UpdateGitPanelLive`）。
- **Problems（診断）**（2026-07）：開いている全エディタタブの診断（LSP `publishDiagnostics` ＋ホスト診断、
  `VimEditorControl.EffectiveDiagnostics`）を集約した一覧。集約自体は `ShellWindow.Problems.cs`（View層。
  各タブの `IEditorLspManager.StateChanged` を購読し、変化のたびに全realizedタブ分を読み直す）が行い、
  `ProblemsPanelViewModel`/`ProblemsPanelView` は表示専用。行クリックで該当ファイル・位置へジャンプする。
  波線（squiggle）表示自体は `Editor.Controls` 側に既に実装済みで、LSPサーバーが繋がっていれば
  このパネルを使わなくても自動で出る——本パネルは「一覧で見える」ことだけを追加する。
- **Settings/Appearance はサイドバーから中央オーバーレイへ移動**（`IsSettingsOverlayOpen`）。左ナビ
  `SettingsCategory`＝外観 / エディタ / ターミナル / AI / 言語サーバー(LSP) / 整形(Formatter) / キーボード の7カテゴリ。
  ⚙＝外観、⌨＝キーボードで開く。同カテゴリ再クリックでトグル閉じ。Esc・背景クリックで閉じる。
  これにより §14（サイドバーで直接編集）は刷新された。**整形**は `:Format` 用 CLI フォーマッタの導入/適用UI
  （言語サーバーカテゴリの双子・§14 末尾参照）。

### 26.3 ステージモード（ソロ＝舞台＋袖＋俯瞰）
> `ShellWindow.Stage.cs` / `ShellWindow.StageCards.cs`。タイトルバーの「ソロ⇄レイアウト」トグル（`OnToggleStageMode`）。

- **舞台（ステージ）**：1ペインを全面に立てる。**袖（ウィング）**：残りの有効ペインを右端に `VisualBrush` の
  ライブミニチュアとして縮小表示する（袖カードは実コントロールを子に持たず、舞台サイズにレイアウト済みの
  非表示ホストを描くだけ＝袖表示のためにペインを動かさない）。**俯瞰（オーバービュー）**：全カードを一望する
  Exposé 風レイヤ（クリックで舞台へダイブ）。
- レイアウトモード（タイル／`PaneHost`）とは**表示の差し替えだけ**で切替わり、レイアウトツリー（`_root`）には
  一切触れない。レイアウトへ戻せば元のタイル配置・比率がそのまま戻る。
- ソロ中に `FocusPane` が呼ばれると対象が自動で舞台に立つ（AI がファイルを開いた・差分を出した等の既存フローが
  そのまま「舞台の自動転換」になる）。
- 有効セッション集合（`_enabledSessions`）はタイル配置（`_root`）と独立し、通常はより広い。Main に出ている有効
  セッションは Main に、出ていない有効セッションは袖に出る（無効はどちらにも出さない）。
- 復元：`StageSnapshot` に Overview を保持（俯瞰を開いたまま離れたら俯瞰のまま戻る・§24.4）。袖の活動バッジは §24.1。
