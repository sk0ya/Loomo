### 設計への影響（重要）
- **TerminalTabView にコマンド実行APIが追加された（sk0ya.Terminal.Controls 1.0.3 以降）** → 解決済み。
  `Task<TerminalCommandResult> RunCommandAsync(string command, CancellationToken)`、
  `TerminalCommandResult(Command, Output, ExitCode, Completed)`、`bool IsShellIntegrationActive` を公開。
  → ただし**エージェントの実行は可視ターミナルへは流さない**。`TerminalService.RunCommandAsync` は常に
    独立した非対話 `Process`（PowerShell）で実行し、AI の出力で人間のターミナルが汚れないようにする。
    可視ターミナルは人間専用。
  → cwd は `cd <dir>` 検知（`TrackChdir`）で追従。`SetWorkingDirectory` は可視ターミナルにも `cd` を反映し、
    フォルダを開いたときに端末が追従するようにする（人間向けの同期で、AI 実行とは独立）。
  （変遷: 1.0.2 以前は実行APIが無く独立 `Process`、1.0.3〜で一時的に可視ターミナルへ一本化、現在は AI 実行を
    ターミナルに流すのをやめて再び独立 `Process` に戻した。）
- **VimEditorControl** は `LoadFile` / `SetText` / `Text` / `FilePath` / `ExecuteCommand(exコマンド)` を公開。
  IEditorService はこれで実装。ただし **選択テキスト取得APIが無い**ため `IEditorService.GetSelectedTextAsync()` は
  現状空文字を返す。FolderTree の現在選択は `get_selection` ツールで取得済み。

### 18. セキュリティ / 安全設計（2026-05-31 実装完了 = §10 の具体化）
- **安全ポリシー（UI非依存）**: `Loomo.Core/Safety`。`SafetySettings`（`AutoApprove` / `RestrictToWorkspaceRoot` /
  `BlockedCommandPatterns`）と `ISafetyPolicy`→`SafetyPolicy`。`SafetyPolicy.Evaluate(toolName, args)` は
  `run_powershell`（旧 `run_command`）のコマンド引数を危険コマンドのブロックリスト（正規表現・大文字小文字無視）と
  照合し `SafetyDecision`(Blocked/Reason) を返す。
  設定インスタンスを保持し評価毎に最新値を読むため**設定変更が即時反映**。
- **既定ブロックパターン**: `rm -rf`（-rf/-fr/分割フラグを lookahead で順不同に検知）・`rmdir /s`・`del /s|/q`・
  `Remove-Item -Recurse -Force`・`format C:` 等のディスクフォーマット（`dotnet format` は除外）・`mkfs`・`dd if=`・
  `> /dev/sd`・`shutdown`・`reboot`・フォークボム。
- **エージェントループ連携**: `AgentOrchestrator` に `ISafetyPolicy` を注入。ツール実行前に `Evaluate`→ブロック時は実行せず
  エラー結果（理由付き）をAIへ差し戻し（`ToolExecutionStarted`/`Completed` も発火しトランスクリプトに表示）。承認は
  `tool.RequiresApproval && !AutoApprove` のときのみ（**自動承認モードで承認カードを省略**）。
- **ワークスペーススコープ制限（パストラバーサル防止）**: `IWorkspaceService.ResolvePath(path)` を追加。`WorkspaceService`
  が `SafetySettings` を見て、`RestrictToWorkspaceRoot` 有効かつルート設定済みのとき、解決後の絶対パスがルート配下
  （`IsWithin`：同一 or `root\` 前方一致, OrdinalIgnoreCase）でなければ `UnauthorizedAccessException`。
  現行の `write_file`/`edit_file`（旧 `open_in_editor`/`propose_edit`）は `ResolvePath` 経由でパスを解決するため、
  相対パス解決と書込スコープ制限を両立する。なお `pwsh` の `Set-Content` 等はこの制限を素通りするため
  ブロックリストのみで守る（CLAUDE.md 参照）。
- **永続化 / UI**: `SafetySettings` は `AiSettings.Safety` に保持し `AiSettingsStore` が平文で保存（秘匿情報ではない）。
  `ApplyTo` は既存インスタンスを書き換え（DIシングルトン参照を保つ）。DI は `AiSettings.Safety` を Singleton 共有 +
  `ISafetyPolicy`/`SafetyPolicy` 登録。設定画面に「自動承認」「ルート配下に限定」チェックボックスを追加。
  危険コマンドのブロックリストは「エディタで編集」ボタンからエディタペインで編集する（§14 の
  `OpenDocumentAsync`／保存コールバック方式。1行1パターン・正規表現、`#` 始まりはコメント）。
  テスト `SafetyTests` 7件（ブロック/許可/非対象ツール/スコープ）パス、計19件。

## 22. ローカルLLM専用化 ＋ Ollama ネイティブAPI移行（2026-06-04 実装 → 2026-06 撤去）

> **⚠️ 撤去済み（§25 に置換）**: 外部 Ollama サーバ前提（`/api/chat`・`OllamaProtocol`/`OllamaClient`/
> `OllamaLauncher`）は **in-process 推論（ONNX Runtime GenAI ＋ llama.cpp）へ全面移行**した。本節は当時の
> 実装記録として残す（現行コードに Ollama 関連は存在しない）。**AIプロバイダの正本は §25**。「ローカルLLM専用・
> クラウド非接続」という決定だけは現在も生きている。
>
> **決定（当時）**: Loomo は **ローカルLLM（Ollama）専用**とする。クラウドAI（Claude / OpenAI / GitHub Copilot）への
> 接続は廃止。狙いはプライバシー・オフライン動作・構成の単純さ。

### 22.1 背景（なぜネイティブAPIへ）
- 旧実装は Ollama の **OpenAI互換エンドポイント**（`/v1/chat/completions`・SSE）を使い、思考量を
  `reasoning_effort`（none/low/medium/high）で送っていた。しかし **qwen3 等は `reasoning_effort` を解釈せず**、
  `none` を指定しても思考（`<think>…</think>`）が出力され続ける問題があった（`reasoning_effort` は実質 gpt-oss 流儀）。
- Ollama **ネイティブ API**（`/api/chat`）は思考の有効・無効を真偽値 `think` で確実に制御でき、思考も
  `message.thinking` として本文と分離して返す。これに全面移行して問題を解消した。

### 22.2 変更点
- **削除**: `OpenAiProtocol`（OpenAI互換ワイヤ処理）・`OpenAiCompatibleClient`、および設計上存在した
  クラウド系クライアント（Claude / Copilot / Stub）と Copilot 認証（§17）。
- **新設**: `Loomo.Ai/Clients/OllamaProtocol`（`/api/chat` のリクエスト組立＋NDJSON ストリーミング）と
  `OllamaClient`（プロバイダ `Local`）。`AiClientFactory` は `OllamaClient` を返す。
- **思考制御**: `ProviderConfig.Thinking`（bool）をそのまま `think` へ写像。thinking 対応モデルのみ送出する
  （`ModelProfiles` が判定。非対応モデルは送らない＝`think:true` でのエラーを回避）。旧 `ThinkingEffort`
  （none/low/medium/high）は `AiSettingsStore` が読み込み時に bool へ移行する。
- **イベント化**: 思考は `message.thinking`→`ThinkingDelta`、本文は `message.content`→`TextDelta`、
  ツールは `message.tool_calls`（`arguments` は**オブジェクト**）→`ToolUseRequested`。`done:true` で完了。
  旧来の `<think>` タグ分離パーサ（`ThinkStreamParser`）は不要となり廃止。
- **モデル一覧・起動確認**: `/v1/models` → **`/api/tags`**（`models[].name`）。`OllamaLauncher` の死活確認も `/api/tags`。
- **ホスト正規化と後方互換**: 既定ホストを `http://localhost:11434/v1` → **`http://localhost:11434`**。
  `OllamaLauncher.ResolveHost` が設定 `BaseUrl` 末尾の `/v1` を自動除去するため、旧 `settings.json` もそのまま動く。
- **ツール非対応モデル**: ネイティブAPIが「does not support tools」を返したら、tools 無しで1回だけ再送する挙動は維持。
