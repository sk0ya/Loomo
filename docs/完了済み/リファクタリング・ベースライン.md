## UI スモークテスト

テスト用ワークスペースには、編集可能なテキストファイルと Markdown ファイル、差分を含む Git リポジトリを用意する。各項目で例外ダイアログが出ず、操作後も入力とフォーカスが継続することを確認する。

| 機能 | 手順 | 期待結果 |
|---|---|---|
| ワークスペース保存・復元 | ワークスペースを開き、複数タブとペインを作って終了し、再起動する | 開いていたワークスペース、タブ、ペイン構成が復元される |
| ペイン分割・移動・表示切替 | ペインを縦横に分割し、フォーカス移動、表示切替、分割解除を行う | 対象ペインだけが変化し、フォーカスと内容が維持される |
| デタッチ・再接続 | ペインを別ウィンドウへデタッチし、そのウィンドウを閉じて再接続する | 内容と状態を失わずメインウィンドウへ戻る |
| 表示モード | ステージ、タイル、俯瞰を順に切り替え、最大化も解除する | 各モードの配置が崩れず、元のペイン構成へ戻れる |
| EditorSupport | Markdown と、利用可能な画像または JSON を開き、通常エディタとの表示を切り替える | プレビューが表示され、リンク、スクロール、再読込が機能する |
| Git | 変更ファイルの差分を開き、テスト用変更をステージしてコミットする | 差分と状態が更新され、コミットが履歴に表示される |
| AI セッション | 新規セッションで短い質問を送り、読み取り専用ツールを1回実行させる | 応答とツール結果が表示され、セッションを再選択できる |

Git コミットと AI ツール実行は作業内容を変更し得るため、使い捨てのテスト用リポジトリで行う。

### Phase 6 自動UIスモーク（2026-07-19）

ビルド済みアプリをWindows UI Automation経由で操作し、次を確認した。

- メインウィンドウ `Loomo` が生成され、UIが応答状態になる
- Automationツリーを取得でき、96個のボタンを含む主要コントロールが生成される
- ステージ→俯瞰→ステージ解除を順に実行しても応答を維持する
- Settings／Explorer／Search／Sessions／Git／Pegboard の各サイドパネルを開閉しても応答を維持する
- 通常終了後に再起動し、名前付きボタン44件と主要UI構成が一致する
- 各実行は `CloseMainWindow` で終了し、終了コード0になる

最終再検証では Release 構成の全1,112テスト（失敗0、スキップ0）を実行後、ビルド済みアプリを
Windows UI Automation で起動した。復元された実ワークスペースを用いて、Markdown の
EditorSupport プレビューが描画されること、Git サイドパネルに現在のブランチと変更件数が表示されること、
Git パネルへ切り替えた後も UI が応答することを確認した。最後に `CloseMainWindow` で終了し、終了コード0を
確認した。スクリーンショットは `artifacts/smoke-git-panel-20260719.png` と
`artifacts/smoke-git-session-20260719.png` に保存した。

当初はプロセス単位の `APPDATA` 差し替えで状態を隔離する想定だったが、WPFの既知フォルダ解決には
反映されず、実ユーザーのワークスペース状態が使われた。追加されたテスト用の重複タブ2件と
デタッチ状態2件は特定して除去し、実行前の15タブ・デタッチなし・アクティブタブ1件へ復元した。
以後の状態変更を伴うスモークは専用ユーザー環境または手動確認で行う。

EditorSupport のデタッチボタンは Automation 上で取得・Invokeできたが、プレビュー対象を開いていない状態では
別ウィンドウが生成されなかったため、デタッチ／再接続成功の証拠には含めない。次の項目は状態変更や外部依存を
含む拡張受け入れ確認として、専用ユーザー、使い捨てGitリポジトリ、導入済みローカルモデルを用意できる場合に
手動確認する（リファクタリング計画の完了ゲートには含めない）。

- ペイン分割・移動・分割解除
- EditorSupport の表示、リンク、スクロール、再読込、デタッチ／再接続
- Git 差分表示、ステージ、コミット、履歴更新
- AI セッション開始、読み取り専用ツール実行、セッション再選択

`ShellWindow` のコンストラクターには14個の依存がある。抽出時の主な境界は次のとおり。

| EditorSupport | `EditorSupport`, `EditorSupportWebView`, `EditorSupportExport`, `Links`, `Browser` | `EditorSupportRegistry`、各 Support、WebView2、テーマ／履歴 | `EditorSupportCoordinator` |
