# Loomo リファクタリング計画

## 1. 目的

Loomo の既存動作を維持しながら、変更が集中している UI 調停処理、巨大な ViewModel、Git 操作、EditorSupport、起動時 DI 構成を段階的に整理する。

主な目標は次のとおり。

- `ShellWindow` に集中した責務を機能単位へ分離する
- UI を必要としないロジックを単体テスト可能にする
- ViewModel からファイル、プロセス、SQLite などの外部操作を分離する
- Git と EditorSupport の機能追加時に巨大クラスを編集せずに済む構造へ移行する
- 現在の `App → Core / Services / AI` という依存方向を維持する

## 2. 現状

### プロジェクト構成

- `Loomo.Core`: UI 非依存のドメイン、エージェント、ツール、抽象
- `Loomo.Ai`: ローカル推論と AI 設定
- `Loomo.Services`: Git、ワークスペース、エディタ、ターミナルなどのアダプタ
- `Loomo.App`: WPF UI、View、ViewModel、UI サービス
- `Loomo.Tests`: Core、Services、App のテスト

- 合格: 1,026件

## 3. 基本方針

1. 各変更では原則として挙動を変えない。
2. 抽出前または抽出と同時に、対象ロジックのテストを追加する。
3. 一度に全面的な MVVM 化を行わず、副作用の少ない純粋ロジックから分離する。
4. 既存インターフェースは Facade として残し、呼び出し側を壊さずに内部を差し替える。
5. 各 PR はレビュー可能な大きさに保つ。
6. 「抽出」「呼び出し切替」「旧実装削除」を必要以上に同じ PR に詰め込まない。

### Phase 1: ShellWindow から状態と純粋ロジックを抽出

期間目安: 3～5日

#### 抽出候補

| コンポーネント | 責務 |
|---|---|
| `PaneLayoutCoordinator` | ペイン分割、移動、表示、フォーカス |
| `StageModeCoordinator` | ステージ、袖、最大化、カード表示 |
| `WorkspaceSessionCoordinator` | ワークスペース保存、復元、切替 |
| `EditorSupportCoordinator` | 対応形式判定、ビュー生成、WebView連携 |
| `DetachedWindowCoordinator` | デタッチ状態とウィンドウ寿命 |
| `CommandPaletteService` | コマンド収集、検索、実行 |

#### 手順

1. 判定や変換などの純粋ロジックを WPF 非依存クラスへ移す。
2. 抽出したロジックにユニットテストを追加する。
3. 既存の `ShellWindow` から新クラスを呼び出す。
4. WPF 要素を扱う調停処理を機能別 Coordinator へ移す。
5. 不要になったフィールド、メソッド、部分クラスを削除する。

#### 完了条件

- `ShellWindow` 部分クラス群が合計6,000行以下になる
- レイアウト判断ロジックを WPF なしでテストできる
- `ShellWindow` はイベント受信と View 更新が中心になる

### Phase 2: 巨大 ViewModel をユースケース単位に分割

#### 対象

- `TrailViewModel`
- `GitSessionViewModel`
- `DiffSessionViewModel`

#### 完了条件

- 原則として各 ViewModel が400行以下になる
- ViewModel にファイル I/O、Git プロセス起動、SQLite 操作が残っていない
- 主要なコマンド処理を UI なしでテストできる

## 6. PR 分割方針

1フェーズを1～3個程度の PR に分割する。

推奨例:

1. 現行ロジックに対する特性テストを追加
2. 新しいクラスを追加し、既存クラスから委譲
3. 呼び出しを切り替え、不要コードを削除

各 PR で確認すること:

- [ ] 挙動変更の有無が説明されている
- [ ] `dotnet test` が成功する
- [ ] 対象機能のスモークテストが成功する
- [ ] 新旧実装の二重管理が残っていない
- [ ] 依存方向を逆転させていない
- [ ] 公開 API の変更が必要最小限である

## 7. 成功指標

- 全1,026テストを継続して成功させる
- `ShellWindow` 関連コードを50%以上削減する
- 500行を超える本番 C# ファイルを原則ゼロにする
- ViewModel から Process、ファイル、SQLite の直接操作を排除する
- 新しい EditorSupport を既存巨大クラスの編集なしで追加可能にする

- `App → Core / Services / AI` の依存方向を維持する

## 8. 対象外

この計画では、次の変更をリファクタリングと同時に実施しない。

- UI デザインの全面変更
- 新機能の追加
- AI モデルや推論バックエンドの変更
- 保存形式の意図的な破壊的変更
- .NET ターゲットフレームワークの更新
- 外部ライブラリの一括更新

必要な場合は、それぞれを独立した計画と PR で扱う。
